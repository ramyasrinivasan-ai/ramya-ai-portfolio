Executive Overview

A Business-First AI System for Recall Readiness

The Ford Recall Readiness Co-Pilot is designed to solve a high-cost enterprise problem: recall-worthy vehicle failure patterns often emerge across fragmented systems long before they become formal recall decisions. Warranty claims, Technical Service Bulletins, NHTSA complaints, engineering notes, and component risk registries may each contain part of the signal, but no single layer synthesizes them into an actionable view fast enough.

This project demonstrates the shift from chatbot thinking to enterprise AI architecture: retrieval, reasoning, confidence scoring, governance, and decision support working together.

The system uses Agentic RAG to retrieve and reason across three evidence layers: a static document corpus, live NHTSA-style complaint signals, and structured warranty claim data. GPT-4o then synthesizes a grounded output with risk level, confidence score, key signals, and a specific recommendation.

01
Business Problem

The Hidden Cost of Reactive Recall Management

The capstone deck frames Ford’s recall challenge around fragmented signals, slow detection, and costly reactive escalation. The current-state problem includes warranty data, NHTSA complaints, TSB patterns, and claim clusters living across disconnected systems, requiring manual analyst effort before a recall readiness decision can be formed.

$3.4B Annual warranty cost cited in the project scenario.
18–24 Months from early signal to recall decision in the reactive model.
50K+ NHTSA complaints received annually in the project framing.
$100M+ Estimated cost exposure per major reactive recall event.

Strategic Interpretation

This is not merely a document-search problem. It is a decision-intelligence problem. The value comes from connecting scattered evidence into a structured recall readiness assessment before regulatory escalation, brand damage, or large-scale remediation costs increase.

02
Why Agentic RAG

Why Traditional ML and Basic Chatbots Are Not Enough

Traditional ML Limitation

Stale and Hard to Explain

  • Traditional ML depends on fixed historical training data.
  • Recall signals evolve continuously as new claims, complaints, and TSB updates appear.
  • Prediction without retrieved evidence is difficult to defend in safety, legal, and engineering contexts.
Agentic RAG Advantage

Current, Grounded, and Traceable

  • Agentic RAG retrieves fresh context at query time.
  • It reasons across multiple evidence layers before generation.
  • Every recommendation can be tied back to source material, confidence, and signal strength.

Design Principle

Recall intelligence requires freshness, explainability, and source grounding. RAG is not chosen because it is trendy; it is chosen because the decision environment changes constantly and every output must be auditable.

03
Architecture

An Azure-Native Agentic RAG Pipeline

Retrieval Foundation

Chunk, Embed, and Index

  • Raw documents are chunked into 512-token segments with overlap.
  • Embeddings are generated using Azure OpenAI embedding models.
  • Azure AI Search stores the vector index and supports semantic retrieval.
Reasoning Layer

Agentic Retriever

  • The agent decomposes user queries into sub-tasks.
  • It retrieves from static documents, live complaint signals, and structured claim data.
  • It ranks passages by relevance, confidence, and recency before synthesis.
Synthesis

GPT-4o Grounded Response

  • The model synthesizes only from retrieved context.
  • If evidence is insufficient, the system should return manual review rather than inventing an answer.
  • This prevents unsupported recall recommendations.
Enterprise Fit

Azure Ecosystem Integration

  • Warranty engineering portal integration through Azure Bot Service.
  • Power BI risk-score tiles for quality dashboards.
  • REST API feed into warranty or ERP systems with Teams/mobile alerts.
04
Data Corpus

Five Synthetic Ford Documents as the Grounding Layer

The project corpus includes synthetic but structurally realistic Ford documents: warranty policy reference, TSB collection, historical warranty claim summary, recall decision framework, and a component risk registry. These provide the domain-specific knowledge base needed for grounded recall reasoning.

Policy Layer

Warranty & Recall Rules

  • Warranty eligibility, coverage duration, exclusions, and claim submission requirements.
  • Recall readiness thresholds such as claim-rate triggers, NHTSA complaint spikes, and similarity to prior recalls.
Evidence Layer

Claims, TSBs, and Component Risk

  • TSB records describe symptoms, root causes, repair procedures, and NHTSA complaint references.
  • Risk registry records active high-watch components, composite scores, trend direction, and threshold distance to recall trigger.
05
Decision Output

From Evidence Retrieval to Recall Recommendation

Structured Output

Every Co-Pilot response is designed to return a repeatable intelligence report: recall readiness risk, trend direction, confidence score, key signals detected, and a specific recommendation for engineering or committee review.

Why This Matters

Recall decisions involve safety, legal exposure, brand trust, and cost. A vague answer is not enough. The system must provide structured decision support that leaders can inspect, challenge, and validate.

Example Decision Logic

A high-risk output may combine rising NHTSA complaints, a matching historical recall pattern, increasing warranty claims, and an active TSB that has not reduced incident rates. The recommendation is then grounded in converging signals rather than a single isolated metric.

06
Validation & Responsible AI

Trust Is Built Into the Architecture

Grounding

Grounded Generation Constraint

  • Responses are restricted to retrieved context.
  • Unsupported claims are blocked by architecture, not just by prompt wording.
  • Low-evidence cases are routed to manual review.
Human Oversight

Engineer-in-the-Loop

  • High-risk recommendations require warranty engineer validation.
  • Overrides and validation outcomes feed back into relevance scoring.
  • Monthly audits compare Co-Pilot recommendations against actual recall outcomes.

Responsible AI Positioning

The system is not designed to replace recall committees or safety engineers. It is designed to compress evidence gathering, improve signal visibility, and make human review faster, more traceable, and more consistent.
